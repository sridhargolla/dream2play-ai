# Support

## Getting Help

For setup, development, and usage questions, start with:

- `README.md` for quick start instructions.
- `USER_MANUAL.md` for product usage guidance.
- `CONTRIBUTING.md` for contribution workflow.
- Existing GitLab issues and discussions for known problems.

## Reporting Bugs

When reporting a bug, include:

- The expected behavior.
- The actual behavior.
- Steps to reproduce.
- Browser and operating system details for frontend issues.
- Node.js and npm versions for backend or tooling issues.
- Relevant logs with secrets removed.

## Requesting Features

Feature requests should describe the user problem, the desired outcome, and any constraints around AI behavior, game generation, or user experience.

## Security Issues

Do not report security vulnerabilities in public issues. Follow `SECURITY.md` for private disclosure guidance.

## Maintenance Expectations

Dream2Play AI is maintained as an open source project. Response times may vary, but well-scoped reports with clear reproduction steps are easier to triage and resolve.
